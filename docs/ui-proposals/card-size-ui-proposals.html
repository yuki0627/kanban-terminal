カードサイズ切替UI — 3方向性の比較

カンバンボードのカード表示サイズを「小・中・大」の3段階で切り替えられるUIを検討中。現状のサイズ(中)は既定値として3案とも維持し、 (1) 切替コントロールをどこに・どんな操作パターンで置くか、(2) サイズごとに何を見せて何を隠すか、の2軸で方向性を分けた3案を並べている。 各タブは実際にクリックして小/中/大を切り替えられる。

A. トップツールバー セグメントコントロール

切替UIはアプリ最上部ツールバーの右端、設定・通知・サウンドの並びに「セグメントコントロール」として常設する。既存 .launcher-btn と同じ 30px 角丸ボタン言語を踏襲しつつ、選択中の項目をアクセントカラーの発光スライダーが追従する構成にした。密度アイコン自体を「行の詰まり方（細い縞＝小／太い縞＝大）」で描き分け、ラベルなしでも直感的に違いが伝わるようにしている。カードは中(M)を基準点に、小(S)は装飾を削り密度を最大化、大(L)はステータス文言・プロジェクトカラーのタグ・メモのプレビューという「普段は隠れている情報」を展開する。

Proposal C

省スペース・サイクル切替アイコン

ツールバー右端のアクション群（通知・サウンド・設定と同列）に、密度を可視化する1個のアイコンボタンを配置する。クリックまたは S キーで「小→中→大」をサイクルし、3択を並べて見せない。アイコン自体の線幅・本数が現在のサイズ状態を表す。 「小」はメモリ・未読マーカーをホバー/フォーカス時のみ開示して密度を最大化し、「大」はプロジェクトチップ・メモ抜粋・ステータス文言を新たに開示する。
