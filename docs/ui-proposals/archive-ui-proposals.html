Archive UI proposals

docs/00-goal.md の確認済み仕様「Archive はレーンとは別のカード属性。archived カードは通常ボードから消え、ボード右上の Archive 一覧から開く・復元できる。Archive まわりの UI は使い勝手を工夫すること」に対する 3 案。いずれもボード右上に入口を置き、 カードを Archive しても他のレーンには影響しない前提は共通。並び順(数字)は優劣ではなく比較のための番号。

1. 右上トレイ + ドロワー

ツールバー右端の Archive アイコン(件数バッジ付き)をクリックすると、ボードを覆わずに右側からドロワーが スライドイン。検索・複数選択・一括 Restore を備えるが、幅は控えめで「ちょっと覗いて戻す」用途に向く。

強み: ボードを離れない・開閉が軽い・実装コストが低い 弱み: 件数が多いとスクロールが長くなる・プレビュー情報が少ない

2. 専用オーバーレイ + 検索・フィルタ

カード展開ウィンドウと同じ視覚言語(オーバーレイ)で、ほぼ全画面の専用ビューを開く。カードはタイル状で memo の抜粋・archived 日時まで見え、Project フィルタとテキスト検索を上部に常設。件数が多くなるプロジェクトの 「昔のカードを探して戻す」用途に強い。

強み: 情報密度が高い・大量カードでも探しやすい・展開ウィンドウと一貫した体験 弱み: 開閉のステップがドロワーよりやや重い・実装コストが中〜大

3. レーン末尾の折りたたみ帯

5 レーンの右端に、Archive を「6 本目のレーンのような帯」として常設する。既定は折りたたみ(薄い縦帯 + 件数)で、クリックで展開するとグレースケールのカードが並ぶ。ドラッグでレーンへ戻す操作も同じ盤面上で完結する — モーダルを開かないぶん「ボードは一枚」という一目性を最も壊さない案。

強み: ボードの一体感を保つ・ドラッグ操作の一貫性・折りたたみ時は場所を取らない 弱み: 展開時にレーン表示が狭くなる・検索/フィルタは別途置き場が必要
