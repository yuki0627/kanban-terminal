# kanban-terminal

kanban-terminal is a local kanban board where every card owns a live terminal.
Cards are created with a shell already running, can be opened instantly, and stay
reattachable through tmux until they are archived.

The board has five work lanes plus an Archive strip. Plain foreground processes
move cards through L2 process detection, and Claude Code activity is detected from
the card PTY without requiring hook injection for hand-typed `claude` commands.

This app runs a local server on `localhost` and does not include authentication.
Use it only on a trusted local machine.

## Install and Run

Requires Node 22.9 or newer, `tmux` for persistent terminals, and the `claude`
CLI on your `PATH` if you want to use Claude Code inside card terminals.

```bash
npx kanban-terminal
```

The default URL is `http://localhost:34567`. If another kanban-terminal server
is already running on that port, a second launch opens the existing server
instead of starting another instance. If the port belongs to another process,
startup stops and asks you to choose a different port.

```bash
npx kanban-terminal --cwd ./my-project
npx kanban-terminal --port 45678
npx kanban-terminal --no-open
```

Global install is supported, but this repository does not publish to npm from CI:

```bash
npm install -g kanban-terminal
kanban-terminal
```

Set `KANBAN_TERMINAL_NO_UPDATE_CHECK=1` or `NO_UPDATE_NOTIFIER=1` to disable the
startup version notice.

## What It Does

- Projects sidebar: add project roots, filter visible projects, and create cards
  under a project or under "Projectなし".
- Card terminals: a card starts a shell immediately, opens by reattaching to that
  live terminal, and survives server restarts through tmux.
- Lane automation: non-shell foreground work moves a card to In Progress, shell
  return moves it to In Review, and Claude PTY activity takes priority while
  Claude is foregrounded.
- Archive: archived cards leave the live board, release their terminal, and can
  be restored by dragging from the Archive strip back to a lane.
- Memory display: total terminal RSS and per-card RSS are shown in the board UI.

## Local Data

kanban-terminal stores app data under:

```text
~/.kanban-terminal/
```

Current files include:

- `board.json`: projects, cards, lanes, archive state, terminal session ids.
- `config.json`: UI settings such as launch commands and attention sound.
- `dev-terminal-sessions.json`: internal terminal session markers.
- `tmux.conf`: the isolated tmux config used by kanban-terminal.

No migration or protection logic for a legacy Electron app `board.json` is
implemented in this branch.

## Development

```bash
yarn
yarn dev
```

In development, Vite serves the client on `http://localhost:6856` and proxies API
and WebSocket traffic to the backend on `http://localhost:34567`.

Required gates:

```bash
yarn format
yarn lint
yarn typecheck
yarn typecheck:server
yarn build
yarn test
```

## Package Contents

The npm package ships:

- `bin/kanban-terminal.js`: launcher for `npx kanban-terminal`.
- `server/`: Express, WebSocket, node-pty, tmux, and board APIs.
- `dist/`: prebuilt Vue client generated by `yarn build`.
