# kanban-terminal02 開発方針(Claude Code 固有)

プロダクトのゴール・UX 原則は `docs/00-goal.md` が SSOT。実装判断に迷ったらそちらを基準にする。
ここには、このリポジトリで開発作業する際の Claude Code 固有の運用ルールだけを置く。

## バグ対応

1. **バグ報告を受けたら、指示が無くても「再現確認 → TDD で修正」を基本方針とする**:
   再現条件を特定(可能なら実データ・実環境で確認) → 失敗するテストを追加(Red) →
   最小修正(Green) → 既存テスト・型チェックで回帰が無いことを確認、の順で進める。
   再現確認だけで原因が特定できない場合は、憶測で修正せず、調査結果を報告してから
   進め方を相談する。
2. 本番相当のローカルデータ(`~/.kanban-terminal/board.json` など)を使って再現確認する場合は、
   作業前に必ずバックアップを取り、実験用に作ったカード・データは作業後に元の状態へ復元する。
